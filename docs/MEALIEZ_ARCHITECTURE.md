# Mealiez Ecosystem Architecture & Migration Plan

This document outlines the strategic refactoring of the Mealiez monorepo into a highly scalable, multi-repository architecture. The goal is to provide a unified API via Nest.js, maintain the Next.js web application for management, and introduce a high-performance Flutter mobile application, all bound together by Supabase Auth and Realtime capabilities.

---

## 1. Multi-Repo Architecture Overview

We are moving from a coupled Next.js application to three distinct repositories.

| Repository Name | Technology Stack | Primary Responsibility |
| :--- | :--- | :--- |
| `mealiez_backend` | Nest.js, Supabase JS, OpenAPI | Central API, Realtime Orchestration, Auth Verification, Database Interaction |
| `mealiez_frontend` | Next.js, React, Tailwind CSS | Admin/Manager Dashboard, Organization Onboarding, Web Client |
| `mealiez_app` | Flutter, Riverpod, Supabase Flutter | Member Mobile App, Native Scanning, Live Tracking, Realtime UI |
| `mealiez_shared` | Zod, TypeScript | Git Submodule containing shared validation schemas and types |

---

## 2. Shared Logic & Types (`mealiez_shared`)

To prevent validation logic from drifting across the backend and frontend, we will extract the existing Zod schemas.

- **Implementation**: Create a Git Submodule named `mealiez_shared`.
- **Contents**: The current `mealiez/lib/validations/` folder.
- **Integration**:
  - `mealiez_frontend` imports it to validate React forms.
  - `mealiez_backend` imports it to validate incoming API requests (using `nestjs-zod`).
  - `mealiez_app` (Flutter) will rely on the OpenAPI spec generated by the backend to guarantee type safety via generated Dart classes.

---

## 3. Backend Implementation (`mealiez_backend`)

The backend will be completely decoupled from Next.js, functioning as a standalone API.

### Core Architecture
- **Framework**: Nest.js (Node.js).
- **Authentication**: Stateless. Nest.js will extract the `Authorization: Bearer <JWT>` header and verify it using the Supabase JWT secret or the `supabase.auth.getUser()` method.
- **Role-Based Access Control**: Implementation of a custom `@Roles('admin', 'manager', 'member')` decorator.

### Migration Priority & Strategy
Move logic from `mealiez/app/api/**` to Nest.js Controllers and Services.
1. **Auth & Onboarding**: Establish system access.
2. **Attendance & Scanning**: Move the core QR logic.
3. **Meal Requests**: Migrate the business logic determining if a meal window is open.

### Supabase Integration
- Use `@supabase/supabase-js`.
- The Nest.js backend will act as the orchestrator for **Realtime events**. For example, when a scan is processed successfully, Nest.js will broadcast a message to a Supabase Channel (e.g., `session_stats`) to instantly update mobile and web dashboards.

### OpenAPI & Code Generation
- Implement `@nestjs/swagger` to generate a comprehensive API contract (`swagger.json`).
- This contract will be used to generate strongly-typed API clients for the Flutter app.

---

## 4. Mobile App Implementation (`mealiez_app`)

The Flutter application will act as a thin, high-performance client optimized for hardware access (camera) and realtime updates.

### Core Architecture & State Management
- **State Management**: **Riverpod** (`flutter_riverpod`). Highly scalable and safe.
- **Routing**: **GoRouter**. Allows declarative routing and auth guards mirroring the Next.js approach.
- **API Client**: **Dio** with an interceptor to inject the Supabase `accessToken` into every request.

### Feature Implementations
- **Realtime Dashboard**: Replace the web app's 10-second polling with Supabase Streams. Use a Riverpod `StreamProvider` to listen to the `attendance` table for instant UI updates when members check-in.
- **Native Scanning**: Replace browser `jsQR` with `mobile_scanner` (MLKit/Vision). This provides vastly superior speed and accuracy for both Admin Scanning and Inventory Barcode scanning.
- **Identity Badge**: Use `qr_flutter` to generate high-quality QR codes and `wakelock_plus` to keep the screen alive when the user displays their badge.
- **Manager Insights**: Utilize `fl_chart` to render beautiful, native pie charts and analytics.

### Styling Strategy
Replicate the existing "Mealiez" Tailwind CSS look in Flutter:
- Define global `ThemeData` specifying custom typography (`GoogleFonts.inter` with heavy font weights for headers).
- Create custom Theme Extensions for specific border radii (`rounded-[2rem]`) and the specific blue/indigo color palette used in the web app.

---

## 5. Frontend Refactor (`mealiez_frontend`)

The Next.js application will transition from a full-stack framework to a pure frontend client.

- **Remove API Routes**: Delete `app/api/**`.
- **Auth Strategy**: Transition from heavy server-side cookie management to a client-side or NextAuth-style JWT approach that correctly passes Bearer tokens to the Nest.js backend.
- **Data Fetching**: Update all components and Server Actions to fetch data from the new Nest.js API endpoint (`NEXT_PUBLIC_API_URL`) instead of querying Supabase directly (where business logic is involved).

---

## 6. Recommended Execution Roadmap

1. **Phase 1: Infrastructure Setup (Week 1)**
   - Initialize the 3 repositories and the Git Submodule.
   - Configure Nest.js with Swagger, Supabase JS, and Auth Guards.
2. **Phase 2: Backend Porting (Weeks 2-3)**
   - Port all critical API routes (Auth, Attendance, Meals) from Next.js to Nest.js.
   - Test endpoints thoroughly using Swagger UI.
3. **Phase 3: Flutter App MVP (Weeks 3-4)**
   - Implement Supabase Auth natively in Flutter.
   - Build the Identity Badge and Realtime Scanner using native plugins.
4. **Phase 4: Web Frontend Cutover (Weeks 4-5)**
   - Refactor Next.js to call the Nest.js backend.
   - Establish Realtime subscriptions in the web dashboard.
5. **Phase 5: Launch & Polish (Week 6)**
   - E2E testing across Web, Mobile, and Backend.
   - Publish mobile apps to stores.
